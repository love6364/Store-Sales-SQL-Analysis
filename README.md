# 🏬 Sales Store Data Analysis Using SQL  
### Advanced SQL Business Intelligence Project

A structured SQL project analyzing retail store sales data to extract business insights, identify top-performing products, and evaluate revenue trends.

---

## 📊 Project Overview

Retail businesses generate large volumes of transactional data.  
However, without proper analysis, it is difficult to understand:

- Which products generate maximum revenue  
- Which customers contribute most sales  
- Seasonal sales patterns  
- Regional performance differences  
- Profitability trends  

This project uses SQL to transform raw sales data into actionable business intelligence.

---

## 🎯 Business Problem

Store management needs answers to key questions:

- What are the top-selling products?
- Which customers generate highest revenue?
- Which categories are most profitable?
- How do sales vary month-by-month?
- Which regions perform best?

Manual spreadsheet analysis is inefficient and not scalable.

This project solves these problems using structured SQL queries.

---

## 📁 Dataset

File Used:
- `sales_store.csv`

The dataset includes:

- Order ID
- Order Date
- Customer Name
- Product Name
- Category
- Region
- Quantity
- Sales
- Profit

---


## 📊 Key Business Insights

- A small number of products contribute majority of total sales.
- Certain regions generate higher profitability.
- Some high-sales products have low profit margins.
- Sales peak during specific months.
- Top customers significantly impact total revenue.

---

## 📈 Business Recommendations

- Focus marketing on high-profit products
- Improve pricing for low-margin high-volume products
- Strengthen strategy in low-performing regions
- Launch loyalty programs for top customers
- Plan inventory based on seasonal demand
